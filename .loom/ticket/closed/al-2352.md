Currently, `loom pack install` can return a non-empty `drifted` list when a managed file already exists on disk but differs from the builtin pack source.

Problem: those conflicting managed paths were not recorded in `.loom/pack/lock.json`, so subsequent `loom pack status` / `loom pack doctor` (and therefore `loom pack status --diff`) would report zero drift and show no diffs.

Fix: on install, still write the pack's expected sha256 for conflicting managed paths into the lock, while continuing to avoid overwriting the repo file unless `--force` is used. This makes drift persistent + inspectable.

Acceptance:
- Install with a preexisting conflicting managed file records that path in lock.json
- status/doctor report drift after install
- status --diff prints diffs for those drifted paths
- Regression tests cover the original mismatch
