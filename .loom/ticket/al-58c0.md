# Modularize team/workspace CLI command groups

## Objective alignment
`team/cli.py` and `workspace/cli.py` are oversized command entrypoints. Public launch needs thin, readable dispatch layers so command behavior is easier to maintain and extend safely.

## Scope
- Refactor:
  - `src/agent_loom/team/cli.py`
  - `src/agent_loom/workspace/cli.py`
- Extract command handlers into grouped modules (e.g., by noun/domain) while keeping parser UX and command names stable.
- Keep top-level CLI files as parser construction + dispatch orchestration only.

## Non-goals
- No command/flag removals or additions unless required for parity-preserving bug fix.
- No behavior changes to command semantics.
- No refactor of non-CLI business logic beyond required import rewiring.

## Implementation plan
1. Define grouping plan for each CLI:
   - Team examples: lifecycle, inbox, objective/sprint, merge.
   - Workspace examples: repo/worktree, harness, cleanup/snapshot.
2. Move command handler functions into grouped modules and wire imports.
3. Retain one canonical parser builder per CLI with unchanged command tree/help text.
4. Ensure JSON mode and text mode output contracts remain identical to pre-refactor behavior.
5. Remove dead command wrappers and unused imports after migration.

## Verification
- `uv run ruff check .`
- `uv run basedpyright`
- `uv run pytest tests/test_team_cli_ux.py tests/test_workspace_cli_ux.py tests/test_workspace_init.py tests/test_workspace_worktree_parity.py`
- `uv run pytest`

## Acceptance criteria
- `team/cli.py` and `workspace/cli.py` are reduced to thin entrypoint responsibilities.
- Command handlers are organized by domain in dedicated modules.
- CLI help trees and command behavior remain stable in tests.
- JSON/text output behavior remains stable.

## Risks / edge cases
- **Risk:** argparse dispatch wiring errors after handler moves.
  - **Detection:** command-not-found or wrong-handler failures in CLI UX tests.
  - **Mitigation:** migrate command groups incrementally and run targeted tests after each group.
- **Risk:** accidental divergence between parser definitions and handler signatures.
  - **Detection:** runtime argument errors and failing UX tests.
  - **Mitigation:** keep parser+handler changes in the same commit slices; avoid deferred wiring.
