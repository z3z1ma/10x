# How Loom Thinks

Treat `.loom/` as the durable work graph, not notes beside the work. Chat is a
shaping channel; records carry recoverable judgment, execution, proof, review, and
reusable knowledge.

## Ambiguity Is A Routing Signal

If intent, scope, constraints, success criteria, evidence, design direction,
system seams, data relationships, state, or coherence are unclear, stay in the
outer loop. Low-record execution is only for tiny, obvious, low-risk work whose
outcome, boundary, non-goals, proof path, and system implications are already
concrete.

## Truth Lives In The Right Surface

Do not let the newest artifact or current chat win by default. Preserve truth in
the surface that owns that kind of truth:

- constitution owns durable project judgment, policy, principles, constraints, ADRs, roadmap
- tickets own executable work units, live state, acceptance, closure posture
- research owns investigation, tradeoffs, rejected paths, null results, synthesis
- specs own intended behavior, requirements, scenarios, interfaces, invariants
- plans own multi-ticket strategy, sequencing, rollout, validation, recovery
- evidence owns observations, outputs, reproductions, screenshots, logs, validation
- audit owns adversarial review findings and verdicts from Ralph review runs
- knowledge owns preferences, procedures, accepted explanations, reusable understanding

When truth is in the wrong place, move the durable version and simplify the
accidental copy.

## Keep The Graph Coherent

Surface authority is typed. A ticket does not rewrite a spec. A ticket-owned Ralph
run does not outrank its ticket or linked records. Evidence proves observations,
not intent. Audit challenges claims; it does not change the product contract. When
surfaces conflict, repair the owning surface and make the conflict visible.

## The Recovery Graph

Records should be easy to find, grep, and understand. Use stable words, IDs,
labels, headings, statuses, dates, and refs. The prose remains the record;
structure helps retrieval.

## Workflow-Specific Skills Move Through Record Skills

Record skills own durable surfaces. Workflow skills add task pressure; they do not
own truth or shorten record, evidence, audit, ticket, or Ralph requirements. If a
workflow would execute before intent, scope, evidence, or ticket boundary is
shaped, stay outer-loop and route the missing truth first.

## Record When It Helps

Always route through Loom. Create or update records when they materially improve
recovery, judgment, execution, verification, review, or reuse. Skip records for
tiny, obvious, low-risk work that leaves no durable truth behind.

## Common Routes

- bounded executable work: tickets
- durable judgment, policy, precedent, constraints, roadmap: constitution
- intended behavior, requirements, interfaces, invariants: specs
- investigation, comparison, rejected paths, null results: research
- multi-ticket sequencing, rollout, recovery: plans
- observations that must support or challenge claims: evidence
- adversarial review through Ralph: audit
- reusable preferences, procedures, explanations, retrieval cues: knowledge
- bounded ticket execution or review: ticket-owned Ralph run
