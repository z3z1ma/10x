# Memory Files Reference

## Cross-Memory Files

- `.loom/memories/hot-memory.md`: top-level current-state pointers across the memory module. Loaded every conversation. <50 lines. Rewrite freely.
- `.loom/memories/link-index.md`: backlink map for memory wiki-links. Auto-generated by housekeeping.
- `.loom/memories/manifest.json`: fixed two-domain manifest for the module.

## System Domain

System is for agent and workflow self-memory: learned patterns, self-observations, and improvements.

- `.loom/memories/system/hot-memory.md`: current system-level priorities and reminders. Rewrite freely. <50 lines.
- `.loom/memories/system/patterns.md`: distilled interaction patterns. Edit in place. Timeless rules only. HARD LIMIT: 70 lines / 5.5KB.
- `.loom/memories/system/self-observations.md`: what worked/didn't. Append-only.
- `.loom/memories/system/improvements.md`: ideas, wishlists, repair notes. Edit in place by section.

## User Domain

User is for non-canonical user context: observations, entities, and supporting reminders.

- `.loom/memories/user/hot-memory.md`: current user-context priorities and reminders. Rewrite freely. <50 lines.
- `.loom/memories/user/observations.md`: raw timestamped events. Append-only. Never edit past entries.
- `.loom/memories/user/entities.md`: 3-line compact registry. People, organizations, named things. Edit in place by `### Name` header.
- `.loom/memories/user/action-items.md`: supporting reminders that are not Loom tickets. Append new, check off done.

For deep recall in either domain, the main warm files are usually:

- `hot-memory.md`
- `observations.md`
- `entities.md`
- `action-items.md`
- any thread files in that same domain

## Glacier

- `.loom/memories/glacier/index.md`: archive catalog. Auto-generated by housekeeping.
- `.loom/memories/glacier/**/*.md`: archived memory snapshots with JSON-compatible frontmatter.

**Retrieval flow:**

1. Read `glacier/index.md` (one small file — the full catalog)
2. Filter by domain/tags/date_range in the table
3. Read only the matching glacier files

## Thread Files

When a topic keeps recurring across observations (3+ observations across 2+ weeks), it can be raised into a **thread** — a read-optimized synthesis file.

Thread files live in their domain directory (e.g., `.loom/memories/user/running.md`).

Every thread has the same spine:

- **Current State** — what's true right now (rewrite freely, always current)
- **Timeline** — dated entries, append-only, full detail preserved (never condensed)
- **Insights** — learnings, patterns, what's different this time

**Rules:**

- **One file forever** — threads grow long, they don't split or condense
- **Texture is the value** — every entry keeps its full detail, quotes, and dates
- **Fragments never move** — threads reference them, don't replace them
- **Current State is always current** — rewrite it freely as things change
