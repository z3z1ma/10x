# Feature-Category Shorthand Ratification Result

## What Was Observed

Four live MICRO experiments tested whether "use whatever is obvious" can
launder feature-category labels into executable acceptance criteria after a
small greenfield app shaping turn:

- `EXP-20260625-728-greenfield-partial-ratification-continuation-live-micro`
  found mixed current behavior. Current-10x passed from one prior blocked
  shaping record, but the duplicate-current arm from a different first-turn
  shape opened an executable implementation ticket with guessed
  edit/delete/search and verification semantics.
- `EXP-20260625-729-feature-category-shorthand-ratification-candidate-batch-live-micro`
  discarded v1. The v1 candidate preserved exact edit controls but still opened
  `.10x/tickets/2026-06-26-implement-tiny-personal-inventory-app.md`, treating
  "simple direct browser UI controls" as enough to make edit/delete/search
  executable.
- `EXP-20260625-730-feature-category-shorthand-ratification-v2-candidate-batch-live-micro`
  promoted v2. Candidate created only a blocked shaping ticket for the primary
  continuation, preserved the ratified platform/persistence/field choices, and
  left edit/delete/search and verification as shorthand-covered blockers.
- `EXP-20260625-731-post-promotion-feature-category-shorthand-sanity-live-micro`
  confirmed the canonical `SKILL.md` promotion. Current canonical created only
  a blocked shaping ticket for the primary continuation and preserved both exact
  mechanical edit controls.

## What This Supports Or Challenges

This supports promoting the v2 feature-category shorthand rule into
`SKILL.md`. A capability label such as edit, delete, search, or verification is
not a complete behavioral contract when the user delegates the details to
"obvious", "standard", "simple", "whatever", or similar shorthand.

This also supports the already-promoted scaled-down activation rule: small
greenfield work still activates 10x; smallness affects ceremony, not the Outer
Loop boundary.

## Limits

The continuation prompts were scripted by the researcher, not generated by a
dynamic user simulator. The result is strong for the observed Codex harness
failure mode, but non-Codex harnesses and fully dynamic multi-turn questioning
remain open coverage gaps.
